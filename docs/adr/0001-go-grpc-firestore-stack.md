# ADR-001: Go + gRPC + Firestore スタック選定

## ステータス

Accepted

## コンテキスト

余剰食品と子ども食堂をマッチングするプラットフォームのバックエンド技術スタックを選定する必要がある。要件は以下の通り:

- リアルタイム性: 食品の消費期限が短いため、低レイテンシの通信が必要
- 型安全性: 複数サービス間（バックエンド/フロントエンド）でのスキーマ共有
- スケーラビリティ: Cloud Run でのサーバーレスデプロイ
- 開発速度: MVP を短期間で完成させる

## 決定

- **バックエンド言語**: Go
- **通信プロトコル**: gRPC + Protocol Buffers（フロントエンドは gRPC-Web）
- **データベース**: Firestore（GCP マネージド NoSQL）

## 理由

### Go
- gRPC エコシステムが最も成熟している言語の一つ
- Cloud Run との親和性（高速起動、低メモリ）
- 静的型付けによるコンパイル時エラー検出

### gRPC + Protocol Buffers
- フロントエンド/バックエンド間の型安全なスキーマ共有
- buf CLI によるスキーマ管理（lint、breaking change 検出）
- gRPC-Web によるブラウザからの直接通信

### Firestore
- スキーマレスで MVP の素早いイテレーションが可能
- リアルタイムリスナー（Phase 2 での通知機能に活用予定）
- GCP IAM との統合認証

## 結果

- Proto 定義がフロントエンド/バックエンド共通の契約として機能
- Firestore エミュレータによるローカル開発環境の簡素化
- gRPC Health Checking Protocol による Cloud Run ヘルスチェック統合
